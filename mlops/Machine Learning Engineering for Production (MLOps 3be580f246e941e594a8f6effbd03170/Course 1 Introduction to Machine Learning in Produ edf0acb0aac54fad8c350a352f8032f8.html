Key challenges of Deployment

concept drift and data drift

data drift: the distribution of the data changes even if the mapping from x to y does not change

concept drift: the desired mapping between x and y changes, it is when the definition of what is y given x changes

concept drift ex. lets say x is the size of a house and y is the price and due to inflation the prices go up and y changes while x stays the same, so the mapping changes.

for data drift it is like people started building bigger or smaller houses so that the distribution of the data changes over time

other issues:

throughput: how many queries per second do you need to handle

logging: to monitor and provide data for the model in the future

Deployment Patterns

ex.

Shadow mode: make the algorithm predict but not take decisions and compare it to the human judgement or an older model

Canary mode: let the algorithm make decisions on a small amount of data, so you can spot problems early on

Blue Green deployment: make the router suddenly switch over to the new green version, it is useful because you can rollback at any time

Degrees of automation: treating the deployment as not a deploy or no deploy decision, do it gradually

partial automation: if the algorithms is not more than 99% sure we leave it for the human

you do not have to get all the way to full automation

human in the loop deployments: making the AI assist only not full automation, sometimes it is the best design point

Skewed Datasets

highly imbalanced datasets

raw accuracy is not useful

build a confusion matrix instead and measure the precision and recall

precision: what fraction did the algorithm get right out of all the predicted examples for a certain class â†’ % from the predicted

recall: what fraction did the algorithm get right from this class (out of the fraction of that class) â†’ % from the ground truth

F1-score: harmonic mean between precision and recall â†’ taking an average but placing more emphasis on the lower number (it is normally very close to the lower number)

these metrics are also useful for multi-class classification
